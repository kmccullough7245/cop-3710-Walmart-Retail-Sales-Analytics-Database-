# COP-3710 - Walmart Retail Sales Analytics Database

## Project Scope
In this project I will design a retail analytics database using Walmart Sales, economic indicators, holidays, and weather data. The database will include time-based partitioning, rolling sales metrics, and triggers to log abnormal sales spikes.  
##  Target Users
- Students and Researcher
- Data Analysts
## Data Sources
Data: [Walmart Dataset](https://www.kaggle.com/datasets/yasserh/walmart-dataset)
## Project Overview
The Walmart Retail Sales Analytics Database is a retail analytics database designed using Walmart weekly sales data combined with economic indicators, holidays, and weather information. The purpose of this database is to support business intelligence reporting, trend analysis, and anomaly detection for retail performance across stores.
This database is designed to support time based partitioning on sales data, rolling sales metrics using SQL window functions, and triggers to log abnormal sales spikes for auditing and monitoring.
## Data Source
This project is based on a Walmart dataset containing weekly sales, store numbers, dates, holiday flags, temperature, fuel price, CPI, and unemployment.
## User Groups
- Retail Analysts
- Store Managers
- Regional Managers
- Business Intelligence Teams
- Data Engineers / Database Administrators
